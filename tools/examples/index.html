Unique Value ¶

This will validate the input to be unique within a set of items.

Options

  • items - {array}, required, the items to validate value against
  • key - {string}, optional, the key to use for each item. Use if items is an array of objects
  • ignore-case - optional attribute, ignore case when checking uniqueness

<input type="text" ng-model="name" items="names" key="'name'" ignore-case helion-unique/>
<input type="text" ng-model="name" items="names" key="'name'" helion-unique/>

Smart Table ¶

A third-party lightweight Angular module to display tabular data with built-in sorting, pagination, filtering and selection. If using the built-in row selection feature, please note that Smart Table will add a isSelected key-value pair to the row object. If the row object originally contained a isSelected key-value pair, it will be overwritten. The code can be viewed at: https://github.com/lorenzofox3/Smart-Table/blob/master/src/stTable.js#L145. It is also recommended to use the st-safe-src attribute so that the original collection is not modified as the table is filtered, sorted, paginated, etc.

Please refer to the website for documentation and more examples: http://lorenzofox3.github.io/smart-table-website

Example 1: A plain read-only table to simply display a list of data.

<table class="table" st-table="exampleCtrl.mockTableDisplayData" st-safe-src="exampleCtrl.mockTableData">
  <thead>
    <tr>
      <th>Name</th>
      <th>Description</th>
      <th>AZ</th>
    </tr>
  </thead>
 <tbody>
   <tr ng-repeat="row in exampleCtrl.mockTableDisplayData">
     <td>{{ row.name }}</td>
     <td>{{ row.description }}</td>
     <td>{{ row.az }}</td>
   </tr>
 </tbody>
</table>

Tabbed Nav ¶

This is a tabbed navigation component that uses UI Router to switch views. Each route should include a `label` and a `state`. The states should be configured in your module: 

$stateProvider.state('route1', {
    url: '/route1',
    views: {
      myView: {
        templateUrl: 'tab-content.html',
        controller: function () {
        }
      }
    }
  });

Options

  • routes - required, the routes (label and state)
  • viewName - required, the view name
var routes = [ { label: 'Route 1', state: 'route1' }, ... ];
    <tabbed-nav routes="routes" view-name="myView"></tabbed-nav>

Percent Gauge ¶

This will add a percent-gauge display to your page. This consists of title and info fields as well as a progress bar. The percent gauge supports two different modes.

Quota Mode

This is the default mode if you don't specify one, or if you specify a mode of 'quota'. It presents the gauge as a quota or usage style - where the user expects to see less than 100% on the gauge. When the gauge shows 100% or more, this is an alert state that the user is made aware of. 

<percent-gauge title="This one's a third" value=".33"></percent-gauge>
<percent-gauge title="This one's a full bar" value="1" value-text="40 / 40 GB"></percent-gauge>
<percent-gauge title="This one's more than the alert threshold" value="1.33"></percent-gauge>

Note: When reaching 100% a "percent-gauge-at-max" class is added to the div child of the percent-gauge; and when exceeding 100% a "percent-gauge-over-max" class is added.

Instance Mode

Specify the mode as "instance" for this mode. It presents the gauge as an instance style - where the user expects to see 100% on the gauge representing the expected state; less than this indicates an alert state. 

<percent-gauge mode="instance" title="Instances as expected" value-text="1/1" value="1"></percent-gauge>
<percent-gauge mode="instance" title="Instances less than expected" value-text="1/3" value=".33" ></percent-gauge>
<percent-gauge mode="instance" title="No instances" value="0"  value-text="0/3"></percent-gauge>

Detail View Service ¶

This is a angular service that can be used to show a right-hand slide-in dialog. You can open multiple detail views and the service will take care of applying styles so that open detail views will stack appropriately. This control uses the Angular Bootstrap dialog framework, so it correctly deals with a scroll bars and supports the ESCape key to close.

The detailView service returns a promise that can be used to hook in methods for when the Detail View is closed or dismissed.

// inject 'helion.framework.widgets.detailView' as 'detailView', and then
detailView({
          title: 'Optional title',
          templateUrl: 'path/to/html/template/for/view.html'
          },
          {
          data: 'second argument is context that is available through the detail view controller'
          }
});

Toast ¶

This is an AngularJS directive that can be used to show toast notifications.

There are 3 standard notification types, a custom type (supporting a custom icon) and a busy indiactor. To use, inject 'helion.framework.widgets.toaster' as 'toaster', then you can use: 

          toaster.success('success message');
          toaster.warniong('warning message');
          toaster.error('error message');
          toaster.show('custom icon message', 'helion-icon-');
          toaster.busy('Busy doing something');

Note that all notifications will automatically dismiss after 5 seconds, apart from the bust toast notification, which will stay shown indefinately - you are repsonsible for dismissing this yourself, using the close() method of the toast object returned. The close() method returns a promise that can be used to chain functionality to be invoked once the busy notification has dismissed, as illustarted below: 

          var toast = toaster.busy('Busy working away');
          .. do something .... maybe async task ....
          toast.close().then(function () {
            toaster.sucess('Completed okay');
          });

Code Block ¶

Shows text in a code block style and allows this to be copied to the user's clipboard for easy use (where supported in the browser).

There are two properties you can set:

  • theme - set to "light" to use the light theme. The default is to use the dark theme.
  • hide-copy - set to true or false to hide or show the copy button. Copy buitton is shown by default.

Dark mode (default) 

<code-block>// This is an example of our code.
You should be able to copy this to your browser if it supports it</code-block>
// This is an example of our code. You should be able to copy this to your browser if it supports it

Light mode 

<code-block theme="light">// This is an example of our code.
You should be able to copy this to your browser if it supports it</code-block>
// This is an example of our code. You should be able to copy this to your browser if it supports it

Without copy button 

<code-block hide-copy="true">// This is an example of our code.
Copy for this block has been disabled</code-block>
// This is an example of our code. Copy for this block has been disabled
