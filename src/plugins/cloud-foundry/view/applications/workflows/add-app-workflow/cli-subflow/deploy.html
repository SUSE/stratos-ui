Deploy Using CLI

You are almost done!

To deploy your app, follow these steps:

1. Download and install the CLI - see here

2. Add a manifest.yml to your source code or update your existing one by copying & pasting the code below. Cloud Foundry uses the manifest file to set the application's name and route. Since the application and its route has been created for you, please ensure that the name, host and domain of your manifest file matches the example below. If the application name and route is not correct in the manifest file, two applications and routes will get created.

--- # We have created a temporary app in the gallery # We have created a delivery pipeline applications: # We have reserved a name/route for your app - name: {{ wizardCtrl.options.userInput.name }} host: {{ wizardCtrl.options.userInput.host }} domain: {{ wizardCtrl.options.userInput.domain.entity.name }} # We have provisioned and bound services services: - {{ service.name }}

3. Enter the following commands in the CLI:


Target the Cloud Foundry API endpoint
cf api {{wizardCtrl.options.userInput.hcfApiEndpoint || '[Cloud Foundry API endpoint]'}}
Login to Cloud Foundry endpoint
$ cf login API endpoint: {{wizardCtrl.options.userInput.hcfApiEndpoint || '[Cloud Foundry API endpoint]'}} Email> {{ wizardCtrl.options.userInput.hcfUserName || '[username]'}} Password> [password] Authenticating... OK
Push to Cloud Foundry endpoint

Navigate to the project directory and execute the following.

cf push {{wizardCtrl.options.userInput.name || '[Application Name]'}}
